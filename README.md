üì± MyPhone Project: How It Works?
This project demonstrates the core principles of Object-Oriented Programming (OOP). Instead of writing everything in a single block, I separated the code into two parts:

The Template (features.java): This is the "blueprint" (like a C struct but smarter). It defines what a phone can do (methods like sentence and space).

The Object (myPhone.java): This is where the magic happens. I created a new instance (object) from the template using the new keyword.

The Logic: The main program sends specific data (Brand and GB) to the object's methods, which then process and display the phone's specifications on the console.
